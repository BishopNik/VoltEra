# ІНК Energy Platform

Преміальна українська платформа для резервного живлення, сонячних систем, інверторів Deye / Anenji / Easun, галереї об'єктів, відгуків, статей, мініфоруму та CRM-адмінки.

## Локальний запуск

```bash
npm install
npm start
```

Якщо використовується `.env.example` з `PORT=4000`, відкрити сайт: `http://127.0.0.1:4000/`

Адмінка: `http://127.0.0.1:4000/admin.html`

Дефолтний dev-вхід:

- логін: `admin`, `Kostia` або `Pasha`
- пароль: `InkAdmin2026!`

Перед публікацією обов’язково змінити пароль через змінні середовища.

## MongoDB

Сервер уже готовий до MongoDB і сам читає локальний файл `.env`. Якщо `MONGODB_URI` не заданий, він працює через локальний fallback `data/db.json`, щоб тестувати без зовнішньої бази. Для MongoDB потрібен встановлений пакет `mongodb` із `package.json`.

Для production створи `.env` або змінні на хостингу:

```bash
PORT=4000
SECRET_KEY=long-random-string-for-signed-admin-session
ADMIN_USER=admin
ADMIN_USERS=admin,Kostia,Pasha
ADMIN_PASSWORD=your-strong-password
MONGODB_URI=mongodb+srv://USER:PASSWORD@CLUSTER.mongodb.net/?retryWrites=true&w=majority
MONGODB_DB=ink_energy
```

`MONGODB_URI` містить логін і пароль користувача MongoDB. `SECRET_KEY` — секрет застосунку для підпису session-cookie адмінки. `ADMIN_USERS` і `ADMIN_PASSWORD` потрібні лише для першого запуску: якщо collection `users` порожня, сервер створює початкові облікові записи із захищеними хешами паролів. Значення `ADMIN_USER` визначає основного адміністратора: він керує користувачами та всіма паролями, а решта акаунтів позначається як «Співробітник компанії» й може змінювати лише власний пароль.

Не надсилай мені пароль у відкритому чаті, якщо не потрібно. Достатньо сказати, що URI готовий, або вставити його в `.env` локально.

## Що реалізовано

- адаптивна головна сторінка українською;
- логотип ІНК з прозорим фоном;
- hero-схема “мережа → панелі → батарея → дім / споживач”;
- м’якша light-тема, менші заголовки секцій, чергування фонів;
- комплекти Base / Pulse / Shift / Orbit з оновленими kW і kWh;
- реальне зображення інтерфейсу Deye Cloud у технологіях;
- контакти з адресою в Солонці, телефонами й картою;
- галерея об'єктів і окрема сторінка галереї;
- відгуки по два в слайдері, видимі кнопки, додавання відгуку через API;
- бейдж “перевірений відгук” показується тільки після CRM-перевірки (`verified=true`) з аудитом, хто змінив статус;
- FAQ у dialog-вікні без зсуву структури;
- Енергоколо з питаннями, окремою сторінкою й API;
- стаття “5 приладів…” відкривається як стаття, а не калькулятор;
- CRM/API/auth адмінка: заявки, відгуки, питання, об'єкти з фото, статті, обладнання;
- механізм непрочитаних лічильників: заявки / відгуки / питання позначаються переглянутими після відкриття розділу;
- опис алгоритмів CRUD: `docs/crud-algorithms.md`;
- схема багатомовності через JS-словники: `docs/i18n.md`;
- SEO: sitemap, robots, Open Graph, canonical, JSON-LD.
